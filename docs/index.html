What you get

Features

Every feature is built for teams that need to pass a security review today, not after their first incident.

SHA-pinned supply chain

Every uses: reference is pinned to a 40-character commit SHA with the human tag kept as a comment. Dependabot rotates them automatically.

Runtime egress firewall

Every job boots with step-security/harden-runner. Audit mode lists every outbound host; switch to block when ready.

Seven languages, zero config

Node, Java (Maven + Gradle), PHP, Python, Go, .NET, Docker. Each job detects its stack and skips in seconds if not present.

Six-tool security mesh

CodeQL · zizmor · gitleaks · Trivy · Checkov · OpenSSF Scorecard. Each tool runs independently and uploads SARIF.

Matrix testing built in

JSON array inputs for Node and Python versions. Coverage artefacts are named per-stack / per-version and aggregated automatically.

Automated releases

release-please reads Conventional Commits, opens release PRs, publishes GitHub Releases, and fast-forwards the sliding v2 tag.

Questions

FAQ

How do consumer repositories receive updates? ▾

Pin @v2. Each release is automatically fast-forwarded on the sliding major tag, and GitHub resolves the new commit SHA on your next CI trigger. No PR is opened in your repository.

What if I only use Python / Node / ...? ▾

Every language job auto-detects its manifest and exits in under a minute if it is not present. Nothing to configure.

Can I lock down outbound network traffic from my CI runners? ▾

Yes. Start with harden-runner-policy: 'audit' to learn the allowlist, then switch to 'block'.
